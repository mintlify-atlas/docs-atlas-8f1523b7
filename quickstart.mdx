# Quick Start Guide

This guide will walk you through your first session with GBI Logistics, from logging in to creating and managing your first transport cube.

<Note>
  This guide assumes you already have the GBI Logistics app installed on your device. If not, see the [Installation Guide](/installation) first.
</Note>

## Prerequisites

Before you begin, ensure you have:

- GBI Logistics app installed on your Android or iOS device
- Employee credentials (username and password)
- Access to the GBI Logistics network (VPN if working remotely)
- A device with camera for barcode scanning (recommended)

## Step 1: First Login

<Steps>
  <Step title="Launch the Application">
    Open the GBI Logistics app from your device. You'll see the login screen with the GBI Logistics logo.
    
    <Info>
      On first launch, the app will check for the minimum required version. If your version is outdated, you'll be prompted to update before logging in.
    </Info>
  </Step>
  
  <Step title="Enter Your Credentials">
    Enter your employee credentials:
    
    - **Usuario (Username)**: Your employee username
    - **Contraseña (Password)**: Your employee password
    
    ```
    Example credentials format:
    Username: jdoe
    Password: ********
    ```
    
    Click the eye icon to toggle password visibility if needed.
  </Step>
  
  <Step title="Authenticate">
    Tap the **"Ingresar"** (Login) button. The app will:
    
    - Validate your credentials against the backend API
    - Retrieve your authentication token
    - Load your dashboard data
    - Navigate you to the main dashboard
    
    <Warning>
      If you see an error message about "versión mínima" (minimum version), your app needs to be updated. Follow the update prompt to download the latest version.
    </Warning>
  </Step>
</Steps>

### Login Screen Reference

The login screen (lib/features/logistics/screens/login_screen.dart:136) includes:

- **Automatic Authentication Check**: If you're already logged in, you'll be redirected to the dashboard automatically
- **Form Validation**: Both fields are required before the login button activates
- **Error Handling**: Clear error messages for invalid credentials or network issues
- **Token Management**: Automatic token storage for future sessions

## Step 4: Creating Your First Transport Cube

Let's create a transport cube to group guides for shipment from customs to warehouse.

<Steps>
  <Step title="Navigate to New Cube Screen">
    From the drawer menu, tap **"Nuevo Cubo"** (New Cube).
    
    You'll see the New Transport Cube screen with:
    - A scan box for adding guides
    - A list of scanned guides (initially empty)
    - A create button at the bottom
  </Step>
  
  <Step title="Scan Guides">
    Use the scan box to add guides to your cube:
    
    1. **Tap the scan input field** or use a barcode scanner
    2. **Scan or enter the guide number** (package code)
    3. **Wait for validation** - the app will:
       - Verify the guide exists
       - Check the guide is in the correct state
       - Add it to the cube if valid
       - Show an error if invalid
    
    <Note>
      Valid guides must be in the "ReceivedInLocalWarehouse" tracking state to be added to a new cube.
    </Note>
    
    Repeat this process for all guides you want to include in the cube.
  </Step>
  
  <Step title="Review Guide List">
    After scanning, review your guide list:
    
    - Each guide shows its package code and current state
    - A counter shows total guides added
    - You can remove individual guides if needed (swipe or tap delete icon)
    
    <Warning>
      You must add at least one guide to create a transport cube. The create button will be disabled if the list is empty.
    </Warning>
  </Step>
  
  <Step title="Create the Cube">
    When you're ready, tap the **"Crear Cubo"** (Create Cube) button at the bottom.
    
    The app will:
    - Send a request to create the cube via API (lib/features/logistics/services/transport_cube_service.dart:18)
    - Assign a unique cube ID
    - Set the initial state to **Created**
    - Navigate you to the Customs Dispatch Details screen
    
    ```dart
    // API Request format
    POST /api/v1.0/Guide/new-transport-cube
    {
      "packageCodes": [
        "PKG001",
        "PKG002",
        "PKG003"
      ]
    }
    ```
  </Step>
  
  <Step title="Confirm Creation">
    You'll see a success message and be taken to the cube details screen where you can:
    
    - View all guides in the cube
    - Add more guides (while in Created state)
    - Remove guides if needed
    - Send the cube to transit when ready
  </Step>
</Steps>

## Step 5: Managing Your Transport Cube

Now that you've created a cube, let's explore what you can do with it.

### Viewing Cube Details

The Customs Dispatch Details screen shows:

- **Cube Header**: Cube ID, state badge, and registration date
- **Guide Counter**: Total number of guides in the cube
- **Guide List**: All guides with their states and details
- **Action Buttons**: State-specific actions (send to transit, add/remove guides)

### Adding More Guides

While in **Created** state, you can add more guides:

<Steps>
  <Step title="Open Scan Box">
    On the cube details screen, use the scan box at the top to add guides.
  </Step>
  
  <Step title="Scan Additional Guides">
    Scan guide barcodes just like during creation. Each valid guide will be added to the cube.
  </Step>
  
  <Step title="Automatic Refresh">
    The guide list updates automatically as you add guides.
  </Step>
</Steps>

### Sending Cube to Transit

When your cube is ready for shipment:

<Steps>
  <Step title="Review Guides">
    Verify all guides are correct and accounted for.
  </Step>
  
  <Step title="Tap Send to Transit">
    Tap the **"Enviar a Tránsito"** button (if available for your role).
  </Step>
  
  <Step title="Confirm Action">
    Confirm the state change when prompted.
    
    The cube will:
    - Change to **Sent** state
    - Move to the "Tránsito a Bodega" screen
    - Become read-only (no more guide additions)
  </Step>
</Steps>

<Info>
  Once a cube is in Sent state, it cannot be modified until it reaches the warehouse and enters the Downloading state.
</Info>

## Step 6: Warehouse Reception Workflow

When cubes arrive at the warehouse, follow this reception process:

<Steps>
  <Step title="Navigate to Warehouse Reception">
    From the drawer menu, select **"Recepción en Bodega"**.
    
    You'll see all cubes in **Downloading** state (or change a Sent cube to Downloading state).
  </Step>
  
  <Step title="Open Cube for Reception">
    Tap a cube to open its details in the Warehouse Reception Details screen.
  </Step>
  
  <Step title="Scan and Verify Guides">
    Use the scan box to verify each guide as it's physically received:
    
    1. Scan the guide barcode
    2. The app validates it belongs to this cube
    3. Visual and audio feedback confirms successful scan
    4. Guide is marked as received
    
    <Note>
      The app provides audio feedback for scan results - listen for success or error sounds.
    </Note>
  </Step>
  
  <Step title="Complete Reception">
    Once all guides are verified, tap **"Completar Recepción"** (Complete Reception).
    
    The cube will:
    - Change to **Downloaded** state
    - Move to "Listo para Entrega" (Ready for Delivery)
    - Be available for client dispatch
  </Step>
</Steps>

## Step 7: Client Dispatch

The final step is dispatching guides to clients:

<Steps>
  <Step title="Navigate to Client Dispatch">
    From the drawer menu, select **"Despacho a Cliente"**.
  </Step>
  
  <Step title="Select Cube">
    Tap a cube in **Downloaded** state to view its details.
  </Step>
  
  <Step title="Dispatch to Client">
    Use the **"Despachar a Cliente"** button to:
    - Mark guides as dispatched to client
    - Update guide tracking states
    - Close the cube if all guides are dispatched
  </Step>
</Steps>

To ensure you have the latest data:

- Individual cube details refresh when opened

## Troubleshooting Common Issues

### Guide Won't Scan

<Warning>
  If a guide won't scan into a cube, check:
  - The guide is in the correct state for the operation
  - The guide isn't already in another cube
  - The barcode is clear and readable
  - You have network connectivity for validation
</Warning>

### Can't Change Cube State

State transitions follow strict rules:

- **Created** → **Sent**: Only if you have dispatch permissions
- **Sent** → **Downloading**: Only at warehouse location
- **Downloading** → **Downloaded**: Only when reception is complete
- Invalid transitions will show an error message

### Session Expired

If you see "Session expired" errors:

1. The app will attempt automatic token refresh
2. If refresh fails, you'll be redirected to login
3. Log in again with your credentials
4. Your work in progress may be lost - save frequently

<Info>
  The app automatically refreshes your authentication token every 15 minutes to prevent session expiration during active use.
</Info>

## Best Practices

### For Efficient Operations

1. **Create Cubes by Route**: Group guides going to the same destination
2. **Verify Before Sending**: Double-check guide counts before sending cubes to transit
3. **Complete Reception Promptly**: Don't leave cubes in Downloading state longer than necessary
4. **Use Audio Cues**: Enable device sound to hear scan success/failure feedback

### For Data Accuracy

1. **Scan, Don't Type**: Use barcode scanning instead of manual entry when possible
2. **Verify Guide States**: Check guide state badges to ensure they're in the expected state
3. **Review Before Confirming**: Always review guide lists before state changes
4. **Report Discrepancies**: If a guide is in an unexpected state, report it immediately
