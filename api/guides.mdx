## Overview

Guides represent individual shipments or packages in the logistics system. This API provides endpoints to list, validate, update, and dispatch guides.

## Get Guides Paginated

Retrieve a paginated list of guides filtered by status.

```http
GET /api/v1.0/Guide/guides-paginated
```

### Query Parameters

<ParamField query="Page" type="integer" required>
  Page number (1-indexed)
</ParamField>

<ParamField query="ItemsByPage" type="integer" required>
  Number of items per page
</ParamField>

<ParamField query="Status" type="string" required>
  Filter by guide status. Common values:
  - `ReadyForDelivery`
  - `InLocalDistribution`
  - `ReceivedInLocalWarehouse`
  - `ReadyForShipment`
  - `DeliveredToFinalDestination`
</ParamField>

<ParamField query="GuideCode" type="string">
  Optional: Filter by specific guide code
</ParamField>

### Request Example

```http
GET /api/v1.0/Guide/guides-paginated?Page=1&ItemsByPage=20&Status=ReadyForDelivery
```

### Response

<ResponseField name="content" type="object">
  Paginated guide list
</ResponseField>

<Expandable title="content properties">
  <ResponseField name="totalRegister" type="integer">
    Total number of guides matching the filter
  </ResponseField>

  <ResponseField name="registers" type="array">
    Array of guide objects
  </ResponseField>

  <Expandable title="registers item">
    <ResponseField name="code" type="string">
      Guide code (e.g., "GBI001234")
    </ResponseField>

    <ResponseField name="subcourierName" type="string">
      Name of the subcourier handling this guide
    </ResponseField>

    <ResponseField name="packages" type="integer">
      Number of packages in this guide
    </ResponseField>

    <ResponseField name="stateLabel" type="string">
      Human-readable current state
    </ResponseField>

    <ResponseField name="updateDateTime" type="string">
      ISO 8601 timestamp of last update
    </ResponseField>
  </Expandable>

  <ResponseField name="userMessage" type="string">
    Optional message for the user
  </ResponseField>
</Expandable>

### Response Example

```json
{
  "code": 0,
  "responseType": "Success",
  "message": null,
  "messageDetail": null,
  "content": {
    "totalRegister": 145,
    "registers": [
      {
        "code": "GBI001234",
        "subcourierName": "Express Delivery Co.",
        "packages": 2,
        "stateLabel": "Listo para Entrega",
        "updateDateTime": "2026-03-06T14:30:00Z"
      },
      {
        "code": "GBI001235",
        "subcourierName": "Fast Logistics",
        "packages": 1,
        "stateLabel": "Listo para Entrega",
        "updateDateTime": "2026-03-06T15:00:00Z"
      }
    ],
    "userMessage": null
  }
}
```

### Flutter/Dart Example

```dart
import 'package:gbi_logistics/features/logistics/services/guide_service.dart';

final response = await guideService.getGuidesPaginated(
  page: 1,
  pageSize: 20,
  status: 'ReadyForDelivery',
  guideCode: null, // Optional filter
);

if (response.isSuccessful && response.content != null) {
  print('Total guides: ${response.content!.totalRegister}');
  for (final guide in response.content!.registers) {
    print('${guide.code}: ${guide.packages} packages - ${guide.stateLabel}');
  }
}
```

## Update Guide Status

Update the status of one or more guides.

```http
POST /api/v1.0/Guide/update-status
```

### Request Body

<ParamField body="guides" type="array" required>
  Array of guide codes to update
</ParamField>

<ParamField body="newStatus" type="string" required>
  New status to apply. Common values:
  - `ReadyForDelivery`
  - `InLocalDistribution`
  - `DeliveredToFinalDestination`
  - `ReceivedInLocalWarehouse`
  - `TransitToWarehouse`
  - `DispatchedFromCustoms`
</ParamField>

### Request Example

```json
{
  "guides": ["GBI001234", "GBI001235", "GBI001236"],
  "newStatus": "InLocalDistribution"
}
```

### Response

```json
{
  "code": 0,
  "responseType": "Success",
  "message": "Status updated successfully",
  "messageDetail": null,
  "content": null
}
```

### Flutter/Dart Example

```dart
import 'package:gbi_logistics/features/logistics/models/operation_models.dart';

final request = UpdateGuideStatusRequest(
  guides: ['GBI001234', 'GBI001235', 'GBI001236'],
  newStatus: 'InLocalDistribution',
);

final response = await guideService.updateGuideStatus(request);

if (response.isSuccessful) {
  print('Updated ${request.guides.length} guides to ${request.newStatus}');
} else {
  print('Error: ${response.messageDetail}');
}
```

## Validate Guide Status by Process

Validate if a guide is in the correct state for a specific process.

```http
POST /api/v1.0/Guide/validate-guide-status-by-process?version=1.0
```

### Request Body

<ParamField body="guideCode" type="string" required>
  Guide code to validate
</ParamField>

<ParamField body="processInformation" type="string" required>
  Process to validate for:
  - `ToRegisterCube` - Validate for cube registration
  - `ToRegisterCubeToDispatch` - Validate for dispatch cube
  - `ToDispatchToClient` - Validate for client dispatch
</ParamField>

<ParamField body="entityRole" type="string">
  Optional: Role of the entity (Courier, Subcourier, FinalClient)
</ParamField>

<ParamField body="entityIdentifier" type="string">
  Optional: Identifier of the entity
</ParamField>

### Request Example

```json
{
  "guideCode": "GBI001234",
  "processInformation": "ToRegisterCube",
  "entityRole": "Courier",
  "entityIdentifier": "COURIER123"
}
```

### Response

<ResponseField name="content" type="object">
  Validation result
</ResponseField>

<Expandable title="content properties">
  <ResponseField name="isValid" type="boolean">
    Whether the guide is valid for the process
  </ResponseField>

  <ResponseField name="currentState" type="string">
    Current state of the guide
  </ResponseField>

  <ResponseField name="requiredState" type="string">
    Required state for the process
  </ResponseField>

  <ResponseField name="message" type="string">
    Validation message
  </ResponseField>

  <ResponseField name="userMessage" type="string">
    User-friendly message
  </ResponseField>
</Expandable>

### Response Example (Valid)

```json
{
  "code": 0,
  "responseType": "Success",
  "message": "Validation successful",
  "messageDetail": null,
  "content": {
    "isValid": true,
    "currentState": "ReceivedInLocalWarehouse",
    "requiredState": "ReceivedInLocalWarehouse",
    "message": "Guide is valid for cube registration",
    "userMessage": "Guía válida para registrar en cubo"
  }
}
```

### Response Example (Invalid)

```json
{
  "code": 0,
  "responseType": "Success",
  "message": "Validation complete",
  "messageDetail": null,
  "content": {
    "isValid": false,
    "currentState": "InLocalDistribution",
    "requiredState": "ReceivedInLocalWarehouse",
    "message": "Guide state does not match required state",
    "userMessage": "La guía no está en el estado correcto"
  }
}
```

### Flutter/Dart Example

```dart
import 'package:gbi_logistics/features/logistics/models/validate_guide_models.dart';

final request = ValidateGuideStatusByProcessRequest(
  guideCode: 'GBI001234',
  processInformation: ValidateGuideProcessType.toRegisterCube,
  entityRole: DispatchRoleType.courier,
  entityIdentifier: 'COURIER123',
);

final response = await validationService.validateGuideStatusByProcess(request);

if (response.isSuccessful && response.content != null) {
  if (response.content!.isValid) {
    print('✓ Guide is valid: ${response.content!.userMessage}');
  } else {
    print('✗ Guide is invalid: ${response.content!.userMessage}');
    print('Current: ${response.content!.currentState}');
    print('Required: ${response.content!.requiredState}');
  }
}
```

## Dispatch Guide to Client

Dispatch one or more guides directly to a client (courier, subcourier, or final client).

```http
POST /api/v1.0/Guide/dispatch-to-client
```

### Request Body

<ParamField body="entityRole" type="string" required>
  Type of recipient:
  - `Courier`
  - `Subcourier`
  - `FinalClient`
</ParamField>

<ParamField body="entityIdentifier" type="string" required>
  Unique identifier of the recipient entity
</ParamField>

<ParamField body="guides" type="array" required>
  Array of guide codes to dispatch
</ParamField>

### Request Example

```json
{
  "entityRole": "Subcourier",
  "entityIdentifier": "SUB-12345",
  "guides": ["GBI001234", "GBI001235"]
}
```

### Response

```json
{
  "code": 0,
  "responseType": "Success",
  "message": "Guides dispatched successfully",
  "messageDetail": null,
  "content": null
}
```

### Flutter/Dart Example

```dart
import 'package:gbi_logistics/features/logistics/models/operation_models.dart';

final request = DispatchGuideToClientRequest(
  entityRole: 'Subcourier',
  entityIdentifier: 'SUB-12345',
  guides: ['GBI001234', 'GBI001235'],
);

final response = await guideService.dispatchToClient(request);

if (response.isSuccessful) {
  print('Successfully dispatched ${request.guides.length} guides');
} else {
  print('Error: ${response.messageDetail}');
}
```

## Move Guide to Different Cube

Move guides from one transport cube to another.

```http
POST /api/v1.0/Guide/change-cube-guide
```

### Request Body

<ParamField body="guides" type="array" required>
  Array of guide codes to move
</ParamField>

<ParamField body="destinationCubeId" type="integer" required>
  ID of the destination cube
</ParamField>

### Request Example

```json
{
  "guides": ["GBI001234", "GBI001235"],
  "destinationCubeId": 1524
}
```

### Response

```json
{
  "code": 0,
  "responseType": "Success",
  "message": "Guides moved successfully",
  "messageDetail": null,
  "content": null
}
```

## Guide Status Values

The following status values are supported:

<AccordionGroup>
  <Accordion title="Initial States">
    - `PackageCreated` - Package has been created in the system
    - `ReadyForShipment` - Ready to be shipped
    - `InAirline` - Package is with the airline
  </Accordion>

  <Accordion title="In Transit">
    - `InternationalTransit` - In international transit
    - `ArrivedInEcuador` - Arrived in Ecuador
    - `TransitToWarehouse` - In transit to warehouse
  </Accordion>

  <Accordion title="Customs">
    - `EnteredCustoms` - Entered customs
    - `DocumentaryInspection` - Under documentary inspection
    - `PhysicalInspection` - Under physical inspection
    - `AutomaticInspection` - Under automatic inspection
    - `InInspectionProcess` - In inspection process
    - `WaitingForWeighing` - Waiting for weighing
    - `Declared` - Declared to customs
    - `DASRejected` - Declaration rejected
    - `AuthorizedExit` - Exit authorized
    - `DispatchedFromCustoms` - Dispatched from customs
    - `DispatchedFromCustomsWithOutCube` - Dispatched without cube
  </Accordion>

  <Accordion title="Warehouse">
    - `ReceivedInLocalWarehouse` - Received in local warehouse
    - `ReadyForDelivery` - Ready for delivery
  </Accordion>

  <Accordion title="Delivery">
    - `InLocalDistribution` - In local distribution
    - `DispatchRequested` - Dispatch requested
    - `DeliveredToFinalDestination` - Delivered to final destination
  </Accordion>

  <Accordion title="Special Cases">
    - `WaitingForTaxPayment` - Waiting for tax payment
    - `SpecialCase` - Special case handling
    - `Observed` - Under observation
    - `EmptyPackage` - Empty package received
    - `DeclaredAbandoned` - Declared as abandoned
    - `DidNotArrive` - Package did not arrive
    - `InShippingCompany` - With shipping company
  </Accordion>
</AccordionGroup>

## Process Information Types

When validating guides, use these process types:

<ParamField body="processInformation" type="string">
  - `ToRegisterCube` - For registering in a transport cube
  - `ToRegisterCubeToDispatch` - For dispatch cube registration
  - `ToDispatchToClient` - For direct dispatch to client
</ParamField>

Each process type validates the guide against different required states.
