## Overview

Transport Cubes are the fundamental organizational units in GBI Logistics for grouping and managing guides through their journey. A transport cube acts as a virtual container that holds multiple guides and tracks them through various stages of the logistics workflow.

## What is a Transport Cube?

A transport cube is a logical grouping mechanism that:
- **Groups related guides** for efficient batch processing
- **Tracks state transitions** through the 4-stage workflow (Created → Sent → Downloading → Downloaded)
- **Enables batch operations** on multiple guides simultaneously
- **Provides traceability** with complete history and audit trail

## Cube Types

## Cube Lifecycle and States

Every transport cube progresses through a 4-stage workflow. See [State Workflow](/features/state-workflow) for detailed state management.

## Cube Information Model

Each transport cube contains key metadata:

```dart lib/features/logistics/models/transport_cube_info.dart
class TransportCubeInfo {
  final int id;                      // Unique cube identifier
  final DateTime registerDateTime;   // Creation timestamp
  final String state;                // Current state (Created|Sent|Downloading|Downloaded)
  final int guides;                  // Number of guides in cube
  final String? stateLabel;          // Human-readable state
  final bool allowClose;             // Whether cube can be closed
  final CubeType type;               // Cube type
  final String? typeLabel;           // Human-readable type
  final String? operatorName;        // Operator who created the cube
}
```

## Creating Transport Cubes

Transport cubes are created through the `TransportCubeService`:

```dart Example: Creating a New Cube
// Prepare the request with guide codes
final request = NewTransportCubeRequest(
  guideCodes: ['GBI001', 'GBI002', 'GBI003'],
  type: CubeType.transitToWarehouse,
);

// Create the cube
final response = await transportCubeService.createTransportCube(request);

if (response.isSuccessful) {
  final cubeId = response.content?.content?.id;
  final labelId = response.content?.content?.transportCubeLabelId;
  print('Cube created: ID=$cubeId, Label=$labelId');
}
```

<Note>
  All guides must pass validation checks before being added to a cube. See [Guide Management](/features/guide-management) for validation details.
</Note>

## Managing Cubes

### Retrieving Cubes

```dart Fetching Transport Cubes
final response = await transportCubeService.getTransportCubes(
  page: 1,
  itemsPerPage: 20,
  state: TransportCubeState.created,
);

if (response.isSuccessful) {
  final cubes = response.content?.items ?? [];
  print('Found ${cubes.length} cubes');
}
```

### Changing Cube State

```dart Transitioning Cube States
final request = ChangeTranportCubesStateRequest(
  transportCubeIds: [101, 102],
  newState: TransportCubeState.sent,
);

final response = await transportCubeService.changeTransportCubesState(request);
```

### Getting Cube Details

```dart Retrieving Detailed Information
final response = await transportCubeService.getTransportCubeDetails(cubeId);

if (response.isSuccessful) {
  final details = response.content;
  print('Cube has ${details.guides.length} guides');
  print('Current state: ${details.state}');
}
```

### Closing Cubes

Cubes must be closed before final dispatch:

```dart Closing Transport Cubes
final response = await transportCubeService.closeTransportCubes([cubeId]);

if (response.isSuccessful) {
  print('Cube closed successfully');
}
```

<Warning>
  Only cubes in the **Downloaded** state with `allowClose: true` can be closed.
</Warning>

## Batch Operations

Transport cubes enable efficient batch processing:

### Dispatch Multiple Cubes to Client

```dart Batch Dispatch
final cubeIds = [101, 102, 103];
final response = await transportCubeService.dispatchCubeToClient(cubeIds);

if (response.isSuccessful) {
  print('${cubeIds.length} cubes dispatched');
}
```

### Moving Guides Between Cubes

```dart Inter-Cube Guide Transfer
final request = ChangeCubeGuideRequest(
  guideCode: 'GBI001',
  sourceCubeId: 101,
  targetCubeId: 102,
);

final response = await transportCubeService.moveGuideBetweenCubes(request);
```

## Best Practices

<AccordionGroup>
  <Accordion title="Cube Size Management">
    - Keep cube sizes reasonable (50-200 guides per cube)
    - Don't mix different cube types
    - Group guides by destination or route
  </Accordion>
  
  <Accordion title="State Transitions">
    - Always follow the linear workflow: Created → Sent → Downloading → Downloaded
    - Validate state transitions before attempting changes
    - Handle failed state transitions gracefully
  </Accordion>
  
  <Accordion title="Error Handling">
    ```dart
    final response = await transportCubeService.createTransportCube(request);
    
    if (!response.isSuccessful) {
      // Check for specific error codes
      if (response.messageDetail?.contains('invalid state') ?? false) {
        // Handle state validation error
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Performance">
    - Use pagination when fetching cube lists
    - Cache cube details to reduce API calls
    - Batch operations when processing multiple cubes
  </Accordion>
</AccordionGroup>
