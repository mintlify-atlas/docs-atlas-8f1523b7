# Welcome to GBI Logistics

GBI Logistics (LogiRuta) is a powerful mobile logistics management application built with Flutter, designed to streamline the entire lifecycle of package dispatch and reception through an innovative **Transport Cube** system.

<Note>
  GBI Logistics is currently at version **1.0.0.8** and actively maintained for Android and iOS platforms.
</Note>

## What is GBI Logistics?

GBI Logistics is a comprehensive SaaS solution that manages the complete logistics workflow from customs clearance to final client delivery. The system uses a unique **Transport Cube** concept - containers that group multiple shipping guides and move through different operational states as they progress through the logistics chain.

### Key Concepts

**Transport Cubes**: Logical containers that group multiple shipping guides together, allowing batch operations and efficient tracking through the logistics pipeline.

**Shipping Guides (Guías)**: Individual packages tracked through various states from customs to final delivery.

**State-Based Workflow**: Both cubes and guides follow a predefined state progression that ensures proper handling and visibility at each stage.

└─────────────┬───────────────────────────┘
              │

              │

### Technology Stack

- **Framework**: Flutter 3.0+ with Dart SDK
- **State Management**: Provider pattern for reactive state
- **API Communication**: RESTful API integration with automatic token refresh
- **Storage**: Flutter Secure Storage for sensitive data, SharedPreferences for user settings

## Transport Cube State Flow

Understanding the state flow is crucial for using GBI Logistics effectively:

<Steps>
  <Step title="Created (Despachado de Aduana)">
    Transport cube is created with selected guides during customs dispatch. This is the initial state where guides are grouped for shipment.
  </Step>
  
  <Step title="Sent (Tránsito a Bodega)">
    Cube is sent from customs to the warehouse. During this state, the cube is in transit and no modifications are allowed.
  </Step>
  
  <Step title="Downloading (Recibido en Bodega Local)">
    Cube arrives at the local warehouse and reception process begins. Guides are verified and received.
  </Step>
  
  <Step title="Downloaded (Listo para Entrega)">
    Reception is complete and guides are ready for final dispatch to clients. This is the final operational state.
  </Step>
</Steps>

## Who Should Use This System?

GBI Logistics is designed for:

- **Logistics Operations Teams**: Managing day-to-day dispatch and reception operations
- **Warehouse Staff**: Receiving and processing incoming transport cubes
- **Customs Personnel**: Creating and dispatching transport cubes from customs
- **Delivery Coordinators**: Managing final dispatch to clients
- **Operations Managers**: Monitoring overall logistics flow and statistics

## Core Screens Overview

### Operational Screens

- **Dashboard**: Real-time statistics and quick access to all operations
- **Customs Dispatch**: Create new transport cubes and manage guides in Created state
- **Warehouse Transit**: Monitor cubes in transit to warehouse (Sent state)
- **Warehouse Reception**: Receive and verify cubes arriving at warehouse (Downloading state)
- **Client Dispatch**: Manage final dispatch of guides to clients (Downloaded state)

### Supporting Screens

- **Transport Cube Details**: View and manage individual cube details across all states
- **Guide Scanner**: Scan and verify individual guides
- **New Transport Cube**: Create new cubes by scanning guides

<Warning>
  Each screen is state-specific and provides operations relevant only to that state. Attempting invalid state transitions will result in errors.
</Warning>

## API Configuration

GBI Logistics connects to the GBI Logistics backend API:

```
// Development Environment
static const String devBaseUrl = 'http://100.104.120.121:81';

// Production Environment  
static const String prodBaseUrl = 'https://internalapi.gbilogistics.net';
```

<Info>
  The application automatically handles token refresh every 15 minutes and will refresh tokens 30 minutes before expiration.
</Info>
